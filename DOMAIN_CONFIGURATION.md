# 微信小程序域名配置说明

## 问题描述
当前小程序尝试请求 `https://apis.map.qq.com` 时出现错误：
```
https://apis.map.qq.com 不在以下 request 合法域名列表中
```

## 解决方案

### 方案一：开发者工具临时设置（推荐用于开发测试）

1. 在微信开发者工具中，点击右上角的"详情"按钮
2. 在"本地设置"标签页中，勾选"不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书"
3. 重新编译项目

**注意：** 此设置仅在开发者工具中生效，真机调试和正式发布仍需要配置合法域名。

### 方案二：配置合法域名（推荐用于正式发布）

#### 在微信公众平台后台配置：

1. 登录微信公众平台：https://mp.weixin.qq.com
2. 进入你的小程序管理后台
3. 点击"开发" -> "开发管理" -> "开发设置"
4. 在"服务器域名"部分，找到"request合法域名"
5. 点击"修改"按钮
6. 添加域名：`https://apis.map.qq.com`
7. 点击"保存并提交"

#### 配置步骤截图说明：
- 进入开发设置页面
- 找到"服务器域名"区域
- 点击"request合法域名"的"修改"按钮
- 在输入框中添加：`https://apis.map.qq.com`
- 保存配置

### 方案三：使用模拟数据（当前已实现）

代码中已经添加了模拟数据作为备选方案：
- 当API请求失败时，会自动使用模拟的篮球场数据
- 模拟数据包含5个篮球场，分布在用户当前位置附近
- 这样可以确保地图功能正常工作，即使没有配置合法域名

2. 检查地图上是否显示真实的篮球场数据
3. 移动地图时，篮球场数据是否动态更新
